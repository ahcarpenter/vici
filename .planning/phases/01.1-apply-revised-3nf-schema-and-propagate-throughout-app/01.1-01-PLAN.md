SQLModel patterns to use (from RESEARCH.md):

TIMESTAMPTZ datetime column:
```python
created_at: datetime = Field(
    default_factory=lambda: datetime.now(UTC),
    sa_column=sa.Column(sa.DateTime(timezone=True), nullable=False),
)
```

FK with ondelete (required for all FKs — Field(foreign_key=) does not support ondelete):
```python
user_id: int = Field(
    sa_column=sa.Column(sa.Integer, sa.ForeignKey("user.id", ondelete="RESTRICT"), nullable=False)
)
```

CHECK + UNIQUE constraints via __table_args__:
```python
__table_args__ = (
    CheckConstraint("pay_rate > 0", name="ck_job_pay_rate_positive"),
    UniqueConstraint("user_id", "window_start", name="uq_rate_limit_user_window"),
)
```

CRITICAL: `user` is a PostgreSQL reserved word. In raw SQL use `"user"` (quoted). SQLAlchemy ORM handles quoting automatically — only text() statements need explicit quotes.

Rewrite migrations/versions/2026-03-05_initial_schema.py in place. Keep the same revision ID and down_revision = None (it remains the root migration). Replace upgrade() and downgrade() entirely.

upgrade() must create tables in FK dependency order:
1. user — id SERIAL PK, phone_hash VARCHAR UNIQUE NOT NULL, created_at TIMESTAMPTZ NOT NULL
2. message — id SERIAL PK, message_sid VARCHAR UNIQUE NOT NULL, user_id INTEGER NOT NULL FK user(id) RESTRICT, body TEXT NOT NULL, message_type VARCHAR NULL, raw_gpt_response TEXT NULL, created_at TIMESTAMPTZ NOT NULL
3. job — id SERIAL PK, user_id FK user(id) RESTRICT, message_id INTEGER NOT NULL UNIQUE FK message(id) RESTRICT, description TEXT NULL, location TEXT NULL, pay_rate FLOAT NOT NULL CHECK(pay_rate > 0), estimated_duration_hours FLOAT NULL CHECK(estimated_duration_hours IS NULL OR estimated_duration_hours > 0), ideal_datetime TIMESTAMPTZ NULL, datetime_flexible BOOLEAN NULL, created_at TIMESTAMPTZ NOT NULL
4. work_request — id SERIAL PK, user_id FK user(id) RESTRICT, message_id INTEGER NOT NULL UNIQUE FK message(id) RESTRICT, target_earnings FLOAT NOT NULL CHECK(target_earnings > 0), target_timeframe TEXT NULL, created_at TIMESTAMPTZ NOT NULL
5. match — id SERIAL PK, job_id FK job(id) RESTRICT, work_request_id FK work_request(id) RESTRICT, matched_at TIMESTAMPTZ NOT NULL; UNIQUE(job_id, work_request_id)
6. rate_limit — id SERIAL PK, user_id FK user(id) RESTRICT, window_start TIMESTAMPTZ NOT NULL, count INTEGER NOT NULL DEFAULT 0; UNIQUE(user_id, window_start)
7. audit_log — id SERIAL PK, message_sid VARCHAR NOT NULL, message_id INTEGER NULL FK message(id) SET NULL, event VARCHAR NOT NULL, detail TEXT NULL, created_at TIMESTAMPTZ NOT NULL

Each table uses sa.ForeignKeyConstraint([col], [target], ondelete="RESTRICT") form for FKs. The message_id FK in audit_log uses ondelete="SET NULL".

Use sa.DateTime(timezone=True) for ALL datetime columns (produces TIMESTAMPTZ in PostgreSQL).

Add appropriate indexes: ix_message_message_sid (unique), ix_message_user_id, ix_user_phone_hash (unique), ix_audit_log_message_sid.

downgrade() drops tables in reverse order: audit_log, rate_limit, match, work_request, job, message, user.

CRITICAL: The `user` table name is a PostgreSQL reserved word. Alembic uses quoted identifiers automatically in op.create_table("user", ...) — this is correct. The migration tooling handles it. No manual quoting needed in Alembic op.* calls.
