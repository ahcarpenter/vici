# Phase 02.10 Plan 01: Wire Temporal TracingInterceptor + sync_pinecone_queue span Summary

**One-liner:** Temporal OTel TracingInterceptor wired via Client.connect interceptors kwarg, plus manual `temporal.sync_pinecone_queue` span with row count and failure event attributes.

## What Was Done

### Task 1: Wire TracingInterceptor in get_temporal_client

Added `from temporalio.contrib.opentelemetry import TracingInterceptor` to `src/temporal/worker.py` and updated `get_temporal_client` to pass `interceptors=[TracingInterceptor(always_create_workflow_spans=True)]` to `Client.connect`. The Worker inherits interceptors from the client automatically — no changes to `run_worker` or `Worker(...)` were needed.

Also fixed two pre-existing E501 ruff violations in worker.py (lines 19 and 57) that were blocking `ruff check`.

### Task 2: Add manual span to sync_pinecone_queue_activity

Wrapped the entire body of `sync_pinecone_queue_activity` in `tracer.start_as_current_span("temporal.sync_pinecone_queue")`. The span:
- Sets `pinecone.rows_fetched` attribute after the SELECT query
- Increments a `failed` counter per exception
- Calls `span.add_event("row_upsert_failed", {"job_id": str(row["job_id"])})` for each failed row
- Sets `pinecone.rows_failed` attribute after the loop

The existing `temporal.process_message` span in `process_message_activity` was not changed.

## Tests Added
