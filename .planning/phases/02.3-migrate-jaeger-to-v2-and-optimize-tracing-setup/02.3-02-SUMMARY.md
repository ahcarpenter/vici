## Decisions Made

- Direct module-level tracer patching in test fixtures instead of global TracerProvider override — `otel_trace.set_tracer_provider()` logs a warning after first provider is set, causing InMemorySpanExporter spans to not be captured reliably across multiple tests
- Twilio span positioned outside/above `asyncio.to_thread` (in async context) — OTel context propagation into threads is not automatic, span must be created in async code and wrap the full `to_thread` await
