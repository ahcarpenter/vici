### Pattern 1: Chained Dependencies with Session Threading
The gate dependencies need access to the same `AsyncSession`. `get_session` must be injected via `Depends` into each gate dependency.

```python
# Source: CLAUDE.md lines 117-130
async def check_idempotency(
    form_data: dict = Depends(validate_twilio_request),
    session: AsyncSession = Depends(get_session),
) -> dict:
    message_sid = form_data.get("MessageSid", "")
    await MessageRepository.check_idempotency(session, message_sid)
    return form_data
```

FastAPI caches dependencies per request — `get_session` called in multiple dependencies returns the same session object. This preserves the existing single-transaction behavior.

### Pattern 2: EarlyReturn Exception for HTTP 200 Paths
Twilio requires HTTP 200 for all responses, including "skip" cases. Using `HTTPException(4xx)` would cause Twilio to retry infinitely.

```python
# src/sms/exceptions.py (extend existing file)
class EarlyReturn(Exception):
    """Raised by dependencies to short-circuit the route with HTTP 200 + empty TwiML."""
    pass

class DuplicateMessageSid(EarlyReturn):
    pass

class RateLimitExceeded(EarlyReturn):
    pass
```

```python
# src/main.py — register exception handler
from fastapi import Request
from fastapi.responses import Response

EMPTY_TWIML = '<?xml version="1.0" encoding="UTF-8"?><Response/>'

async def early_return_handler(request: Request, exc: EarlyReturn) -> Response:
    return Response(content=EMPTY_TWIML, media_type="text/xml")

app.add_exception_handler(EarlyReturn, early_return_handler)
```

The audit log write for early-return paths belongs in the dependency itself (before raising), per D-06. This means `check_idempotency` and `enforce_rate_limit` dependencies need access to `AuditLogRepository` — achievable by injecting `session` into each dependency.

### Pattern 3: UserRepository Extraction
`MessageRepository.get_or_create_user` is a pure user domain operation. Extract verbatim to `users/repository.py`:

```python
# src/users/repository.py (new)
class UserRepository:
    @staticmethod
    async def get_or_create(session: AsyncSession, phone_hash: str) -> User:
        ...  # same logic as current MessageRepository.get_or_create_user
```

`MessageRepository.get_or_create_user` should then delegate to `UserRepository.get_or_create` or be removed. The new `get_or_create_user` dependency in `sms/dependencies.py` imports from `users` domain, not `sms`.

### Anti-Patterns to Avoid
- **Raising HTTPException(4xx) for Twilio early returns:** Returns 4xx → Twilio retries → duplicate processing. Always return 200.
- **Direct private attribute access on collaborators:** `self._service._client` breaks encapsulation and makes testing harder. Use public properties.
- **Empty stub files:** CLAUDE.md says add files only when they contain real logic. No empty `service.py`, `router.py`, etc.

## Common Pitfalls

### Pitfall 1: Breaking the Single Transaction
**What goes wrong:** Splitting the gate dependencies across multiple `session.begin()` contexts or using separate sessions, losing atomicity.
**Why it happens:** Each Depends() call looks independent, but all must share the same session within one transaction.
**How to avoid:** Inject `Depends(get_session)` into every gate dependency. FastAPI returns the same session instance for all deps in one request. Keep `async with session.begin()` in the router (or rely on the existing `get_session` context manager that yields a session with autobegin).
**Warning signs:** Tests showing partial writes when one gate fails.

### Pitfall 2: AuditLogRepository Import in Dependencies
**What goes wrong:** Circular import — `sms/dependencies.py` importing `sms/audit_repository.py` which imports from `sms/models.py`, creating a cycle if `sms/router.py` also imports from both.
**Why it happens:** The audit write for early-return paths must move into the dependency per D-06, adding a new import.
**How to avoid:** Audit imports are one-directional: `dependencies.py` → `audit_repository.py` → `models.py`. This chain is safe. Verify with `ruff check src` after the change.
**Warning signs:** `ImportError: cannot import name X` on startup.

### Pitfall 3: EarlyReturn Handler Missing from `create_app()`
**What goes wrong:** `EarlyReturn` is raised in a dependency but no handler is registered, causing FastAPI to return 500.
**Why it happens:** Exception handler must be registered after `app = FastAPI(...)` in `create_app()`.
**How to avoid:** Add `app.add_exception_handler(EarlyReturn, early_return_handler)` alongside the existing `TwilioSignatureInvalid` handler in `create_app()`.
**Warning signs:** Tests expecting 200 response for duplicate SIDs return 500.

### Pitfall 4: TwiML Constant Duplication
**What goes wrong:** `EMPTY_TWIML` constant defined in both `sms/router.py` and the exception handler.
**Why it happens:** The exception handler needs the constant but is registered in `main.py`.
**How to avoid:** Define `EMPTY_TWIML` in `sms/router.py` (or `sms/constants.py`) and import it into the handler, OR define the handler in `sms/` domain and import it into `main.py`.

### Pitfall 5: Test Fixture Compatibility
**What goes wrong:** Tests that previously exercised the inline gate logic in the router no longer reach it because the gates are now dependencies — test may need to mock the dependency.
**Why it happens:** FastAPI test client respects Depends() — if the dep is not overridden, it will call the real DB.
**How to avoid:** Review `tests/sms/` and `tests/test_repositories.py` after refactor. Use `app.dependency_overrides` for unit tests. Integration tests should still exercise the full chain.
