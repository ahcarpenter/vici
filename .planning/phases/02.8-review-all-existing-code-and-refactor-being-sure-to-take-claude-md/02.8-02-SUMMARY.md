## What Was Built

### src/users/repository.py (new)
`UserRepository.get_or_create(session, phone_hash)` extracted verbatim from `MessageRepository.get_or_create_user`. Per D-08: repositories own their domain.

### src/sms/exceptions.py (refactored)
`EarlyReturn` base exception added. `DuplicateMessageSid` and `RateLimitExceeded` now subclass `EarlyReturn` instead of `Exception`. `early_return_handler()` and `EMPTY_TWIML` co-located here. Handler returns HTTP 200 TwiML — Twilio never sees 4xx.

### src/sms/dependencies.py (extended)
Three new chained gate dependencies:
- `check_idempotency` — wraps idempotency SELECT in `session.begin()` to avoid autobegin leak; writes audit on duplicate, then re-raises
- `get_or_create_user` — chains from `check_idempotency`; upserts user in its own `session.begin()` block
- `enforce_rate_limit` — chains from `get_or_create_user`; rate-limit check + audit in separate `session.begin()` blocks

### src/sms/router.py (thinned)
Route body reduced to: extract fields from gates tuple -> persist message + audit -> emit Inngest event. All gates removed from route body. `Request` import removed. `EMPTY_TWIML` imported from `sms.exceptions` (single source of truth).

### src/main.py
`app.add_exception_handler(EarlyReturn, early_return_handler)` registered after `TwilioSignatureInvalid` handler.
