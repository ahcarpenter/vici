<decisions>
## Implementation Decisions

### Pydantic Field() constraints
- **D-01:** Add `Field()` constraints to **all** schemas — both HTTP-boundary schemas (e.g., `TwilioWebhookPayload`) and internal DTOs (e.g., `JobCreate`, `WorkRequestCreate`, extraction schemas).
- **D-02:** Apply appropriate constraints: `gt=0` for positive integers/floats, `min_length=1` for required strings, `max_length` where reasonable, `ge=0` for zero-inclusive numerics.
- **D-03:** Update test fixtures as needed if validation rejects previously-accepted values.

### `sms/router.py` dependency refactor
- **D-04:** Full `Depends()` refactor per CLAUDE.md. Idempotency, user get/create, and rate limit enforcement extracted into dependency functions.
- **D-05:** Early-return HTTP 200 paths (duplicate message, rate limit exceeded) handled via a custom exception class + FastAPI exception handler — not by raising `HTTPException(4xx)`. Twilio requires 200 responses; 4xx causes Twilio retries.
- **D-06:** Audit log writes for early-return paths belong in the exception handler or the dependency itself, not inline in the route.
- **D-07:** The route body shrinks to: parse payload → emit Inngest event. All pre-flight checks are dependencies.

### Domain structure — missing files
- **D-08:** Extract `User` lookup/creation logic from `MessageRepository` into a proper `users/repository.py`. `MessageRepository` should not own user operations.
- **D-09:** Do NOT create empty `service.py` or `dependencies.py` stubs for domains that have no HTTP routes. Files are added only when they contain real logic.
- **D-10:** `matches/` domain is left entirely to Phase 3. No changes to `src/matches/` in this phase.

### Bug fix — `retry_count` vs `attempts`
- **D-11:** Fix `inngest_client.py:137` — SQL references `retry_count` but the `PineconeSyncQueue` model column is `attempts`. Change to `attempts=attempts+1`.

### Inline imports
- **D-12:** Move all function-level imports to module level: `import time` in `extraction/service.py:33`, `from datetime import timezone` in `jobs/repository.py`.

### Database index naming
- **D-13:** Add `POSTGRES_INDEXES_NAMING_CONVENTION` dict to `database.py` per CLAUDE.md standard. Apply to the SQLAlchemy `MetaData` instance.
- **D-14:** Existing implicit indexes (via `Field(index=True, unique=True)`) do not need to be renamed retroactively via migration — this would be destructive. The convention applies to new indexes going forward.

### Encapsulation — private attribute access
- **D-15:** `extraction/orchestrator.py` accesses `self._extraction_service._client` and `self._extraction_service._settings` directly. Expose these via public properties or pass them as constructor parameters to `PineconeClient`.

### Claude's Discretion
- Exact `max_length` values for string fields where no business rule dictates a limit (use reasonable defaults: 1000 for descriptions, 255 for short strings)
- Whether to use `model_validator` or field-level validators for cross-field constraints
- Exact structure of the custom early-return exception class name/hierarchy

</decisions>
