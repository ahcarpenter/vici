# Phase 02.11 Plan 01: Edge-Case Hardening Summary

Hardened all existing features against 13 identified edge cases. All fixes are surgical one-to-five line changes with defensive guards, fail-fast validation, and correct error signaling.

## What Was Done

### Task 1: Config validation, GPT None guard, activity missing-message error

**src/config.py** — Added `_validate_required_credentials` model_validator that raises `ValueError` at startup if `database_url`, `twilio_auth_token`, `openai_api_key`, or `pinecone_api_key` is empty. App fails fast before binding the port.

**src/extraction/service.py** — Added `timeout=30.0` to OpenAI `.parse()` call. Added None-guard: if `completion.choices[0].message.parsed is None`, raises `ApplicationError("GPT returned unparseable response", non_retryable=False)`.

**src/temporal/activities.py** — Replaced `return "ok"` in the `if message is None:` branch with `raise ApplicationError(..., non_retryable=True)`. Temporal will not retry permanent failures.

### Task 2: Handler catches + rate limit rolling window

**src/pipeline/handlers/unknown.py** — Wrapped `asyncio.to_thread(twilio.messages.create, ...)` in try/except. On failure: logs `unknown_reply_failed` and returns. Activity does not crash.

**src/pipeline/handlers/job_posting.py** — Wrapped pinecone_sync_queue INSERT in its own try/except. On INSERT failure: logs `pinecone_queue_insert_failed`. Job record is already committed; failure is isolated.

**src/sms/repository.py** — Replaced calendar-minute truncation (`replace(second=0)`) with rolling 60-second window using `timedelta(seconds=60)`. Each message gets its own timestamp row; COUNT(*) queries the last 60 seconds. Note: a migration to drop the UNIQUE constraint on `(user_id, created_at)` is needed before deploying.

### Task 3: Shutdown timeout + webhook field validation

**src/main.py** — Replaced `await _worker_task` with `await asyncio.wait_for(_worker_task, timeout=10)` catching both `CancelledError` and `TimeoutError`. Added `consecutive_failures` counter to `_update_gauges()`: after 5 consecutive DB failures, logs at CRITICAL and sets gauge to -1.

**src/sms/dependencies.py** — Added guard in `validate_twilio_request` after form parsing: if `MessageSid` or `From` is missing/empty, raises `HTTPException(status_code=400)`.

**tests/conftest.py** — Added `OPENAI_API_KEY` and `PINECONE_API_KEY` to test env defaults to satisfy new startup validation.

### Task 4: hash_phone guard, datetime parse warning, sync failure log

**src/sms/service.py** — Added `if not e164_number: raise ValueError(...)` guard at top of `hash_phone`.

**src/jobs/repository.py** — Added `structlog.get_logger().warning("job.ideal_datetime_parse_failed", raw_value=...)` in the `except (ValueError, TypeError)` block.

**src/temporal/activities.py** — Added `if failed: logger.warning("sync-pinecone-queue: sweep completed with failures", processed=..., failed=...)` after the sweep loop.
