## Decisions Made
- Used RPCError with ALREADY_EXISTS status code for cron workflow idempotency since WorkflowAlreadyStartedError is not exported from temporalio.client in current SDK version
- Preserved module-level singleton pattern (_orchestrator, _openai_client) from inngest_client.py for activities -- set by run_worker before worker.run()
- ProcessMessageWorkflow retry policy uses maximum_attempts=4 (equivalent to inngest retries=3: 1 initial + 3 retries)
- deployment.environment OTel attribute now derived from settings.env != "production" instead of settings.inngest_dev
