**New pitfalls to add (from STATE.md Accumulated Context):**

| Pitfall Name | What Goes Wrong | Prevention | Source Decision |
|---|---|---|---|
| Twilio sync SDK blocks async event loop | `client.messages.create()` called directly in async def blocks asyncio | `asyncio.to_thread()` wrapper required | Phase 02-gpt-extraction-service |
| Module-level vars for Inngest DI | Inngest functions can't receive injected deps directly; module-level `_orchestrator` var set by lifespan is cleanest pattern | Set module vars in FastAPI lifespan, not at import time | Phase 02.1-03, Phase 02.5 |
| OTel ALWAYS_ON vs ParentBasedTraceIdRatio | ParentBasedTraceIdRatio causes ambiguous sampling based on upstream parent decision; traces can go missing | Use ALWAYS_ON sampler for unambiguous local coverage | Phase 02.3 |
| OpenSearch replicas=0 for single-node | replicas > 0 causes yellow cluster health on single-node OpenSearch; Jaeger index creation fails | Set replicas=0 in index template for local dev | Phase 02.3 |
| OTel context not propagated into threads | `asyncio.to_thread()` breaks OTel context propagation; span created in main thread isn't parent of work inside thread | Create span in async context wrapping the `to_thread` call, not inside the sync function | Phase 02.3 |
| Circular imports with metrics | Importing Prometheus metrics at module top-level in service.py causes circular imports through the DI graph | Import metrics inside the function that uses them (lazy import) | Phase 02.4-01, Phase 02.5 |
| jaeger_query in collector extensions list | `jaeger_query` is not a valid extension for standalone jaeger-collector; listing it causes startup failure | Only list valid collector extensions; jaeger-query is a separate service | Phase 02.3 |
| Async Alembic env.py | async engine requires `asyncio.run()` wrapping in `env.py` — standard synchronous Alembic env.py will error | Use asyncio.run() + conn.run_sync(do_run_migrations) pattern | Phase 01-01 |
| expire_on_commit=False required | Async SQLAlchemy cannot lazy-load after commit; accessing attributes after commit raises error | Set `expire_on_commit=False` on async_sessionmaker | Phase 01-01 |
| Rate limit SELECT bypasses ORM cache | ORM identity cache can return stale counts for rate limit SELECT in same session | Use raw SQL for rate limit SELECT to bypass identity cache | Phase 02.1-01 |
| Inngest Function not directly callable in tests | `process_message(ctx)` raises error; the wrapper is not callable | Call `process_message._handler(ctx)` in tests | Phase 02-gpt-extraction-service |
| Patch target for wrapped OpenAI client | `braintrust.wrap_openai` patch target misses if service.py uses direct import; patch `src.extraction.service.wrap_openai` | Patch at the import site in the module under test, not at the source module | Phase 02-gpt-extraction-service |
| HEALTHCHECK must use liveness endpoint | Health probe that checks DB connectivity will fail if DB is unreachable during startup; causes container restart loop | `/health` must be liveness only — no DB dependency | Phase 02.5 |

### Accurate Architecture

```
SMS Webhook (POST /webhook/sms)
  └── 5-gate security chain:
        1. Twilio signature validation (HTTPException 403 on failure)
        2. MessageSid idempotency check (unique constraint)
        3. User get-or-create (phone_hash)
        4. Rate limit check (raw SQL, bypasses ORM cache)
        5. Persist message to messages table
        └── Inngest event emit (message.received) → HTTP 200 to Twilio

Inngest process-message function (3 retries, on_failure handler → pipeline_failures_total)
  └── PipelineOrchestrator.run()
        ├── Job branch: ExtractionService → JobRepository.create() flush → write_job_embedding() → Pinecone
        ├── WorkRequest branch: ExtractionService → WorkRequestRepository.create() flush → single commit
        └── Unknown branch: asyncio.to_thread(twilio_client.messages.create) → graceful SMS reply

Inngest sync-pinecone-queue cron (*/5 * * * *)
  └── Sweeps pinecone_sync_queue (max 50/run), retries failed embeddings

DI Graph (FastAPI lifespan):
  AsyncOpenAI → wrap_openai (Braintrust) → ExtractionService
  ExtractionService + repos + pinecone_client + TwilioClient → PipelineOrchestrator
  PipelineOrchestrator → inngest_client._orchestrator (module-level var)
  wrap_openai → inngest_client._openai_client (module-level var)
```

# JobRepository.find_matching() — earnings math SQL
# Pattern: rate × estimated_duration ≥ target_earnings
# NULL handling: jobs with NULL pay_rate excluded; NULL estimated_duration flagged
# Sort: ORDER BY ideal_datetime ASC NULLS LAST, estimated_duration ASC NULLS LAST
# Limit: 3–5 results

SELECT id, description, pay_rate, estimated_duration, ideal_datetime, location
FROM jobs
WHERE pay_rate IS NOT NULL
  AND estimated_duration IS NOT NULL
  AND pay_rate * estimated_duration >= :target_earnings
ORDER BY ideal_datetime ASC NULLS LAST, estimated_duration ASC NULLS LAST
