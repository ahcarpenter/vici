## What Was Built

### DRY fix: GPT_MODEL single source of truth

`ExtractionSettings.gpt_model` default was a separate hardcoded copy of `"gpt-5.3-chat-latest"` in `src/config.py`. `ExtractionService._call_with_retry` was also importing `GPT_MODEL` directly from `constants.py`, bypassing the injected `self._settings`.

Changes:
- `src/config.py` now imports `GPT_MODEL` from `src.extraction.constants` and uses it as `_DEFAULT_GPT_MODEL` for `ExtractionSettings.gpt_model`
- `src/extraction/service.py` removes the `GPT_MODEL` import; `_call_with_retry` uses `self._settings.extraction.gpt_model`
- Return annotation on `_call_with_retry` corrected to `tuple[ExtractionResult, Any]`

### Naming fix: utils.py

`src/extraction/pinecone_client.py` was renamed to `src/extraction/utils.py` via `git mv` (preserving history) to conform to the AGENTS.md domain structure convention. Import sites in `src/main.py` and `src/temporal/activities.py` updated accordingly.

### SRP fix: _update_gauges

`_update_gauges` was a nested coroutine defined inside the `lifespan` async context manager. It has no closure dependencies so there was no reason for nesting. Moved to module-level immediately before `lifespan`, and the `from src.metrics import pinecone_sync_queue_depth` import moved to the top-level imports section.
