  <name>Task 3: Extract _update_gauges from lifespan to module-level (SRP)</name>
  <files>
    src/main.py
  </files>
  <action>
    `_update_gauges` is currently a nested coroutine defined inside the `lifespan` async
    context manager in `main.py`. This is an SRP violation: the lifespan function should
    orchestrate startup/shutdown, not define implementations inline.

    The function has no closure dependencies — it accesses only `get_sessionmaker()` (imported
    at module level) and `pinecone_sync_queue_depth` (a module-level singleton from `src.metrics`).

    **Move `_update_gauges` to module level**:

    Current location (nested inside `lifespan`):
    ```python
    from src.metrics import pinecone_sync_queue_depth

    async def _update_gauges():
        consecutive_failures = 0
        while True:
            ...
    ```

    New location: immediately before the `lifespan` function definition, as a module-level
    async function. Move the `from src.metrics import pinecone_sync_queue_depth` import to
    the top-level imports section of `main.py` (alongside the other `src.metrics` import if
    any, or add it with the other `src` imports).

    The `_gauge_task = asyncio.create_task(_update_gauges())` line in lifespan remains unchanged.

    The extracted function:
    ```python
    async def _update_gauges() -> None:
        """Poll pinecone_sync_queue every 15 s and update the depth gauge."""
        consecutive_failures = 0
        while True:
            try:
                async with get_sessionmaker()() as session:
                    result = await session.execute(
                        text(
                            "SELECT COUNT(*) FROM pinecone_sync_queue"
                            " WHERE status = 'pending'"
                        )
                    )
                    pinecone_sync_queue_depth.set(result.scalar_one())
                    consecutive_failures = 0
            except Exception as exc:
                consecutive_failures += 1
                if consecutive_failures > 5:
                    structlog.get_logger().critical(
                        "gauge_updater: repeated DB failures, metric unreliable",
                        consecutive_failures=consecutive_failures,
                        error=str(exc),
                    )
                    pinecone_sync_queue_depth.set(-1)
                else:
                    structlog.get_logger().warning(
                        "gauge_updater: pinecone_sync_queue depth read failed"
                        " — metric stale",
                        error=str(exc),
                    )
            await asyncio.sleep(15)
    ```
