## Evidence

  implication: This salt is bound to Pulumi Cloud's encryption key, not to any passphrase. No passphrase will ever work with this salt.

## Resolution

root_cause: The stack secrets in Pulumi.dev.yaml (encryptionsalt and all `secure:` values) were encrypted using Pulumi Cloud's built-in encryption service. After commit f0e4198 switched the backend to GCS, Pulumi tries passphrase-based decryption, but the encryptionsalt is from Pulumi Cloud's encryption — no passphrase will match. The encrypted values need to be re-encrypted using passphrase-based encryption. Same issue affects staging and prod stack configs.
fix: For each stack, run `pulumi stack change-secrets-provider passphrase` to generate a new encryptionsalt, then re-set any secret config values with `pulumi config set --secret`. This replaces the Pulumi Cloud-encrypted salt and ciphertext with passphrase-encrypted equivalents.
