## What Was Built

**render.yaml** — Render Blueprint defining a Web Service (Docker runtime, `uv run alembic upgrade head` pre-deploy) and a Managed PostgreSQL database (`vici-db`, Postgres 16). DATABASE_URL is injected via `fromDatabase`. All secrets use `sync: false` requiring manual dashboard setup before first deploy.

**.env.example** — Complete env var documentation covering all fields in `src/config.py` plus Docker Compose vars (POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD) and Inngest Cloud vars documented in comments with source instructions.

**.github/workflows/ci.yml** — Single-job CI pipeline: checkout → setup-uv (with cache) → `uv sync --frozen` → `ruff check src/ tests/` → `pytest tests/ -x --tb=short -q`. Uses SQLite for test isolation (no postgres dependency in Actions).

**README.md** — Inngest Cloud Setup section covering account creation, INNGEST_SIGNING_KEY (Manage -> Signing Key), INNGEST_EVENT_KEY (Manage -> Event Keys -> Create Key), app URL registration, and local dev server usage.
