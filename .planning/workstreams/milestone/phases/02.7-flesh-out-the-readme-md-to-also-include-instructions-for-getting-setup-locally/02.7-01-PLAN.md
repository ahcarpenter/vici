**# Vici**

One-sentence description: "Vici is an SMS-driven job matching platform — workers text an earnings goal and receive a ranked list of jobs that let them hit that goal in the shortest possible time."

**## How It Works**

Brief 3-4 sentence narrative covering the core flow:
- Workers text in an earnings goal (e.g. "I need $500 this week")
- Job posters text in a job listing with pay rate and duration
- GPT classifies and extracts structured data from both
- The system matches workers to jobs via earnings math and replies via Twilio SMS

**## Tech Stack**

Bullet list: Python 3.12, FastAPI, SQLModel + Alembic (PostgreSQL), OpenAI GPT (via Braintrust), Pinecone (vector store), Inngest (background jobs/retries), Twilio (SMS), OpenTelemetry -> Jaeger v2 (OpenSearch), Prometheus + Grafana, Docker Compose, Render.com (production).

**## Prerequisites**

List exactly:
- Docker and Docker Compose
- Python 3.12+ and [uv](https://docs.astral.sh/uv/) (`pip install uv`)
- Accounts with API keys for: Twilio, OpenAI, Pinecone, Braintrust
- (Optional) Inngest Cloud account — only needed for production; local dev uses the bundled Inngest Dev Server

**## Local Setup**

Numbered steps:

1. Clone the repository
   ```
   git clone <repo-url>
   cd vici
   ```

2. Copy the example env file and fill in your secrets (see Environment Variables below)
   ```
   cp .env.example .env
   ```

3. Start the full stack (Postgres, OpenSearch, Jaeger, Prometheus, Grafana, Inngest Dev Server, and the API)
   ```
   docker compose up
   ```
   The API is available at http://localhost:8000. The Inngest Dev Server UI is at http://localhost:8288.

4. Apply database migrations (first time and after schema changes)
   ```
   uv run alembic upgrade head
   ```
   Note: Migrations run automatically on container startup via the Dockerfile CMD, so this step is only needed when running the API outside Docker.

**## Deployment**

The project deploys to Render.com via the `render.yaml` Blueprint:
1. Push to `main` triggers a Render deploy (web service + managed PostgreSQL)
2. Set all production env vars in the Render dashboard (see Environment Variables table above)
3. Set `INNGEST_DEV=0` and configure `INNGEST_SIGNING_KEY` / `INNGEST_EVENT_KEY` from Inngest Cloud
4. Render runs migrations automatically on startup via the Dockerfile CMD
