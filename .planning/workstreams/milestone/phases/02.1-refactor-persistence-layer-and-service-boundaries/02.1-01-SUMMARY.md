### Auto-fixed Issues

**1. [Rule 1 - Bug] MockSettings in test files used flat attribute names**

- **Found during:** Task 2
- **Issue:** Three test files (tests/extraction/test_persistence.py, tests/extraction/test_service.py, tests/inngest/test_process_message.py) had `MockSettings` classes with flat `openai_api_key`, `twilio_auth_token` etc. attributes that broke when `ExtractionService.__init__` and `inngest_client.py` were updated to use nested paths.
- **Fix:** Added nested sub-model attributes (`extraction`, `sms`, `pinecone`, `observability`) to each MockSettings class while preserving flat attributes for backward compatibility.
- **Files modified:** tests/extraction/test_persistence.py, tests/extraction/test_service.py, tests/inngest/test_process_message.py
- **Commit:** 36092da

**2. [Rule 1 - Bug] SQLAlchemy ORM identity cache returned stale rate_limit count**

- **Found during:** Task 3
- **Issue:** `enforce_rate_limit` read the rate_limit row via ORM `select(RateLimit)` after a raw SQL upsert. The identity map returned the stale (pre-upsert) ORM instance, causing the count to always be 1 regardless of how many times the function was called in the same session.
- **Fix:** Replaced the ORM SELECT with a raw `text("SELECT count FROM rate_limit ...")` query, which bypasses the identity cache and always returns the live value from the DB.
- **Files modified:** src/sms/repository.py
- **Commit:** 7400ac1
