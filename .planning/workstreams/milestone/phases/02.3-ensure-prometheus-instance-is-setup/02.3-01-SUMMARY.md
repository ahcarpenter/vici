## Decisions Made

- Metrics imported lazily inside `process()` (not at module top) to prevent circular imports in test environments that create multiple service instances.
- `inngest_queue_depth` is a stub Gauge always at 0 — Inngest dev server has no public metrics API.
- `_call_with_retry` now returns `(result, usage)` tuple so token counts are available to `process()` without re-querying the API.
