**GKE Ingress (per APP-04, D-03 only FastAPI app):**
- Use annotation `kubernetes.io/ingress.class: gce` (NOT ingressClassName field -- not reliable on GKE per RESEARCH.md)
- Set `kubernetes.io/ingress.allow-http: "true"` (required for HTTP-01 ACME challenge)
- Set `cert-manager.io/issuer: letsencrypt-staging` (switch to `letsencrypt-prod` post-verify per D-14)
- TLS: secret_name=`vici-tls`, hosts=[APP_HOSTNAME]
- Rules: host=APP_HOSTNAME, path="/", pathType="Prefix", backend service name="vici-app" port name="http"
- depends_on: [tls_secret_placeholder, staging_issuer, prod_issuer]

```python
vici_ingress = k8s.networking.v1.Ingress(
    "vici-ingress",
    metadata=k8s.meta.v1.ObjectMetaArgs(
        name="vici-ingress",
        namespace="vici",
        annotations={
            "kubernetes.io/ingress.class": _INGRESS_CLASS,
            "kubernetes.io/ingress.allow-http": "true",
            "cert-manager.io/issuer": "letsencrypt-staging",
        },
    ),
    spec=k8s.networking.v1.IngressSpecArgs(
        tls=[k8s.networking.v1.IngressTLSArgs(
            secret_name=_TLS_SECRET_NAME,
            hosts=[APP_HOSTNAME],
        )],
        rules=[k8s.networking.v1.IngressRuleArgs(
            host=APP_HOSTNAME,
            http=k8s.networking.v1.HTTPIngressRuleValueArgs(
                paths=[k8s.networking.v1.HTTPIngressPathArgs(
                    path="/",
                    path_type="Prefix",
                    backend=k8s.networking.v1.IngressBackendArgs(
                        service=k8s.networking.v1.IngressServiceBackendArgs(
                            name=_APP_SERVICE_NAME,
                            port=k8s.networking.v1.ServiceBackendPortArgs(name=_APP_SERVICE_PORT),
                        ),
                    ),
                )],
            ),
        )],
    ),
    opts=ResourceOptions(
        provider=k8s_provider,
        depends_on=[tls_secret_placeholder, staging_issuer, prod_issuer],
    ),
)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet -> GKE Load Balancer -> Ingress -> Pod | Public HTTPS endpoint; TLS terminated by cert-manager certificate |
| cert-manager -> Let's Encrypt ACME | Automated certificate issuance via HTTP-01 challenge |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Spoofing | TLS certificate | mitigate | cert-manager provisions certificates from Let's Encrypt CA; ACME HTTP-01 challenge validates domain ownership |
| T-05-07 | Tampering | Ingress routing | mitigate | Single Ingress rule routes only to vici-app Service; no other backends exposed (D-03, D-04) |
| T-05-08 | Information Disclosure | Internal UIs | mitigate | Temporal UI, Grafana, Jaeger remain ClusterIP-only per D-03/D-04; no Ingress rules for observability namespace |
| T-05-09 | Denial of Service | ACME rate limits | accept | Use staging issuer first (D-14); production issuer rate limit of 50 certs/domain/week is acceptable for this scale |
</threat_model>
