**.github/workflows/cd-base.yml** (reusable workflow per D-05):
```yaml
name: CD Base

on:
  workflow_call:
    inputs:
      stack:
        required: true
        type: string
      command:
        required: true
        type: string
      environment:
        required: false
        type: string
    secrets:
      WIF_PROVIDER:
        required: true
      WIF_SERVICE_ACCOUNT:
        required: true
      PULUMI_ACCESS_TOKEN:
        required: true

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: ${{ inputs.environment }}
    permissions:
      contents: read
      id-token: write

    steps:
      - uses: actions/checkout@v4

      - uses: google-github-actions/auth@v3
        with:
          workload_identity_provider: ${{ secrets.WIF_PROVIDER }}
          service_account: ${{ secrets.WIF_SERVICE_ACCOUNT }}

      - name: Configure Docker for Artifact Registry
        run: gcloud auth configure-docker us-central1-docker.pkg.dev --quiet

      - name: Build and push image
        if: inputs.command == 'up'
        run: |
          PROJECT_ID=$(gcloud config get-value project)
          IMAGE="us-central1-docker.pkg.dev/${PROJECT_ID}/vici-images/vici"
          SHA=$(git rev-parse --short HEAD)
          docker build -t "${IMAGE}:${SHA}" -t "${IMAGE}:${{ inputs.stack }}" .
          docker push "${IMAGE}:${SHA}"
          docker push "${IMAGE}:${{ inputs.stack }}"

      - uses: astral-sh/setup-uv@v5
        with:
          enable-cache: true

      - name: Install Pulumi dependencies
        run: cd infra && uv sync --frozen

      - uses: pulumi/actions@v6
        with:
          command: ${{ inputs.command }}
          stack-name: ${{ inputs.stack }}
          work-dir: infra
        env:
          PULUMI_ACCESS_TOKEN: ${{ secrets.PULUMI_ACCESS_TOKEN }}
```

Notes on cd-base.yml:
- `if: inputs.command == 'up'` on the build step skips image build during `preview` (staging PRs per D-09)
- `astral-sh/setup-uv@v5` matches existing `ci.yml` pattern for Python dependency management
- `uv sync --frozen` in `infra/` installs Pulumi Python SDK dependencies
- Image tagging: SHA + env tag per D-06

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions OIDC -> GCP WIF -> CI SA | Keyless authentication; trust is scoped to repository owner attribute condition |
| CI SA -> Artifact Registry | Image push access; tampered images could deploy malicious code |
| CI SA -> GKE cluster | Deployment access; over-permissioned SA could modify unrelated resources |
| CI SA -> GCS state bucket | Pulumi state access; state contains resource IDs and some config values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Spoofing | GitHub OIDC token | mitigate | WIF attribute_condition scopes to `assertion.repository_owner == 'ahcarpenter'`; only this org's repos can authenticate |
| T-05-11 | Tampering | Docker image in Artifact Registry | mitigate | Only `vici-ci-push` SA has `roles/artifactregistry.writer`; all other access is read-only |
| T-05-12 | Elevation of Privilege | CI SA roles | mitigate | SA has minimum roles: `artifactregistry.writer`, `storage.objectAdmin` (state bucket only), `container.developer` (K8s deploy only); no `roles/owner` or `roles/editor` |
| T-05-13 | Elevation of Privilege | Prod deployment without approval | mitigate | cd-prod.yml requires `environment: prod` with GitHub required reviewers gate (CD-03) |
| T-05-14 | Information Disclosure | Static GCP credentials in GitHub | mitigate | No static SA keys; WIF OIDC provides short-lived tokens only (CD-04) |
| T-05-15 | Tampering | Pulumi state manipulation | accept | `storage.objectAdmin` grants broad state bucket access; acceptable because CI SA is the only writer and state bucket is env-scoped |
</threat_model>
