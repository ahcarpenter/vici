<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External traffic -> GKE Ingress -> FastAPI pod | Untrusted HTTP requests from the public internet |
| Pod -> Cloud SQL via Auth Proxy | IAM-authenticated DB access via Unix socket |
| Pod -> GCP Secret Manager via ESO | Workload Identity scoped to namespace |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | FastAPI Deployment | mitigate | Pod runs as `vici-app` KSA with Workload Identity binding to app GSA; no other SA can impersonate |
| T-05-02 | Tampering | Container image | mitigate | Image pulled from private Artifact Registry; only `vici-ci-push` SA has write access (registry.py) |
| T-05-03 | Information Disclosure | Environment variables | mitigate | All secrets injected via ESO-generated K8s Secrets (envFrom); no secrets in Deployment spec or container args |
| T-05-04 | Denial of Service | FastAPI pods | mitigate | HPA auto-scales 1-3 replicas on CPU 70%; GKE Autopilot enforces resource limits (250m/512Mi request) |
| T-05-05 | Elevation of Privilege | Cloud SQL Auth Proxy | mitigate | Auth Proxy runs as non-root (UID 65532); uses IAM authentication only, no password |
</threat_model>
