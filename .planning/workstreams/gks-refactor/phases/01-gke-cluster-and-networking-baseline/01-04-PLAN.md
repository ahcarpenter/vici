  <name>Task 1: Provision Dev Environment and Verify Idempotency</name>
  <files></files>
  <action>
Run `pulumi up --stack dev` and then run the idempotency check. These commands provision the actual GCP infrastructure.

**Step 1: Set environment variables and run first `pulumi up`**

```bash
cd infra
export PULUMI_BACKEND_URL="gs://vici-pulumi-state-dev"
export PULUMI_CONFIG_PASSPHRASE="<your passphrase>"
pulumi up --stack dev --yes
```

Expected output pattern:
```
Updating (dev)...

     Type                                     Name                   Status
 +   pulumi:pulumi:Stack                      vici-infra-dev         created
 +   ├─ gcp:container:Cluster                 vici-cluster           created
 +   ├─ gcp:serviceaccount:Account            app-gsa                created
 +   ├─ gcp:serviceaccount:Account            ci-push-sa             created
 +   ├─ gcp:serviceaccount:IAMBinding         app-wi-binding         created
 +   ├─ gcp:artifactregistry:Repository       vici-registry          created
 +   ├─ gcp:artifactregistry:RepositoryIamMember  ci-push-iam        created
 +   ├─ kubernetes:core/v1:Namespace          ns-vici                created
 +   ├─ kubernetes:core/v1:Namespace          ns-temporal            created
 +   ├─ kubernetes:core/v1:Namespace          ns-observability       created
 +   ├─ kubernetes:core/v1:Namespace          ns-cert-manager        created
 +   └─ kubernetes:core/v1:Namespace          ns-external-secrets    created

Resources:
    13 created

Duration: ~10-20 minutes (GKE Autopilot cluster creation takes 7-12 minutes)
```

Note: GKE Autopilot cluster creation typically takes 7–12 minutes. This is normal.

**Step 2: Run idempotency check (must show zero changes)**

```bash
cd infra
export PULUMI_BACKEND_URL="gs://vici-pulumi-state-dev"
export PULUMI_CONFIG_PASSPHRASE="<your passphrase>"
pulumi preview --stack dev --expect-no-changes
```

Expected output:
```
Previewing update (dev)...

Resources:
    13 unchanged

exit code 0
```

If this command exits with code 1, it means Pulumi is proposing changes — this is a failure. Common causes:
- `dns_config` not set explicitly in cluster.py (check: the field must be in the resource body, not in ignore_changes)
- A new field was added to ignore_changes that should not be there
- GCP mutated a field that is not yet guarded

Do not proceed to namespace verification if idempotency check fails. Diagnose and fix cluster.py first.

**Step 3: Retrieve cluster credentials**

```bash
# Get the cluster name and location from Pulumi outputs
cd infra
export PULUMI_BACKEND_URL="gs://vici-pulumi-state-dev"
CLUSTER_NAME=$(pulumi stack output cluster_name --stack dev)
CLUSTER_LOCATION=$(pulumi stack output cluster_location --stack dev)
GCP_PROJECT=$(pulumi config get gcp:project --stack dev)

# Configure kubectl
gcloud container clusters get-credentials "$CLUSTER_NAME" \
  --location="$CLUSTER_LOCATION" \
  --project="$GCP_PROJECT"
```

<task type="checkpoint:human-verify">
  <name>Task 2: Verify Namespaces and Artifact Registry</name>
  <what-built>
Task 1 provisioned the full dev environment: GKE Autopilot cluster, five Kubernetes namespaces, Artifact Registry repository, and GCP service accounts. Task 1 also verified idempotency via `pulumi preview --expect-no-changes`.

This checkpoint verifies the Kubernetes and registry layers directly.
  </what-built>
  <how-to-verify>
**Step 1: Verify all five namespaces exist**

```bash
kubectl get namespaces
```

Expected output includes all five namespaces (plus default Kubernetes system namespaces):
```
NAME                STATUS   AGE
cert-manager        Active   Xm
default             Active   Xm
external-secrets    Active   Xm
kube-node-lease     Active   Xm
kube-public         Active   Xm
kube-system         Active   Xm
observability       Active   Xm
temporal            Active   Xm
vici                Active   Xm
```

Confirm: cert-manager, external-secrets, observability, temporal, vici — all five are Active.

**Step 2: Verify Artifact Registry repository exists**

```bash
cd infra
export PULUMI_BACKEND_URL="gs://vici-pulumi-state-dev"
REGISTRY_URL=$(pulumi stack output registry_url --stack dev)
echo "Registry URL: $REGISTRY_URL"

# List repositories in the project
GCP_PROJECT=$(pulumi config get gcp:project --stack dev)
gcloud artifacts repositories list \
  --location=us-central1 \
  --project="$GCP_PROJECT"
```

Expected: vici-images repository listed with format DOCKER.

**Step 3: Test Artifact Registry push using CI service account credentials**

```bash
# Impersonate the CI service account to test push access
CI_SA_EMAIL=$(pulumi stack output ci_push_sa_email --stack dev)

# Authenticate Docker to Artifact Registry using application default credentials
gcloud auth configure-docker us-central1-docker.pkg.dev

# Pull a minimal test image and push it to verify write access
docker pull hello-world:latest
docker tag hello-world:latest "${REGISTRY_URL}/hello-world:test"
docker push "${REGISTRY_URL}/hello-world:test"
```

Expected: Push succeeds. You should see layer digests and "latest: digest: sha256:..." in the output.

If push fails with 403 PERMISSION_DENIED: the IAM binding may not have propagated yet. Wait 2–3 minutes and retry — GCP IAM changes can take up to 7 minutes to propagate.

**Step 4: Clean up test image**

```bash
gcloud artifacts docker images delete \
  "${REGISTRY_URL}/hello-world:test" \
  --project="$GCP_PROJECT" \
  --quiet
```

**Step 5: Confirm Workload Identity is enabled on the cluster**

```bash
CLUSTER_NAME=$(pulumi stack output cluster_name --stack dev)
CLUSTER_LOCATION=$(pulumi stack output cluster_location --stack dev)
gcloud container clusters describe "$CLUSTER_NAME" \
  --location="$CLUSTER_LOCATION" \
  --project="$GCP_PROJECT" \
  --format="value(workloadIdentityConfig.workloadPool)"
```

Expected output: `<PROJECT_ID>.svc.id.goog`
