  <name>Task 1: Manual Bootstrap — Enable GCP APIs and Create GCS State Buckets</name>
  <what-built>Nothing yet — this task documents required manual setup that cannot be automated by Pulumi because Pulumi needs its state backend to exist before it can run.</what-built>
  <how-to-verify>
Run each command block below for each environment. Replace PROJECT_ID_DEV, PROJECT_ID_STAGING, PROJECT_ID_PROD with the actual GCP project IDs for each environment (vici-dev, vici-staging, vici-prod).

**Step 1: Enable required GCP APIs in each project**

```bash
# dev
gcloud services enable \
  container.googleapis.com \
  artifactregistry.googleapis.com \
  cloudresourcemanager.googleapis.com \
  iam.googleapis.com \
  secretmanager.googleapis.com \
  storage.googleapis.com \
  --project=PROJECT_ID_DEV

# staging
gcloud services enable \
  container.googleapis.com \
  artifactregistry.googleapis.com \
  cloudresourcemanager.googleapis.com \
  iam.googleapis.com \
  secretmanager.googleapis.com \
  storage.googleapis.com \
  --project=PROJECT_ID_STAGING

# prod
gcloud services enable \
  container.googleapis.com \
  artifactregistry.googleapis.com \
  cloudresourcemanager.googleapis.com \
  iam.googleapis.com \
  secretmanager.googleapis.com \
  storage.googleapis.com \
  --project=PROJECT_ID_PROD
```

**Step 2: Create GCS state buckets (one per environment, versioning enabled)**

```bash
gsutil mb -p PROJECT_ID_DEV -l us-central1 gs://vici-pulumi-state-dev
gsutil versioning set on gs://vici-pulumi-state-dev

gsutil mb -p PROJECT_ID_STAGING -l us-central1 gs://vici-pulumi-state-staging
gsutil versioning set on gs://vici-pulumi-state-staging

gsutil mb -p PROJECT_ID_PROD -l us-central1 gs://vici-pulumi-state-prod
gsutil versioning set on gs://vici-pulumi-state-prod
```

Note: GCS bucket names are globally unique. If any name is taken, use `vici-{initials}-pulumi-state-{env}` and update the backend URL in Pulumi.yaml and the CI environment variable accordingly.

**Step 3: Install prerequisites on this machine**

```bash
# Install Pulumi CLI
brew install pulumi
# or: pip install pulumi

# Install gcloud CLI (if not present)
# https://cloud.google.com/sdk/docs/install

# Install gke-gcloud-auth-plugin
gcloud components install gke-gcloud-auth-plugin

# Authenticate to GCP
gcloud auth application-default login
```

**Step 4: Verify all three buckets exist**

```bash
gsutil ls gs://vici-pulumi-state-dev
gsutil ls gs://vici-pulumi-state-staging
gsutil ls gs://vici-pulumi-state-prod
```

Expected output: no error (empty bucket listing is fine).

**infra/Pulumi.yaml**

```yaml
name: vici-infra
description: Vici GKE infrastructure — all environments
runtime: python
backend:
  url: gs://vici-pulumi-state-dev   # default for local dev; CI overrides via PULUMI_BACKEND_URL
```

Note: The backend URL here is the local dev default only. CI pipelines must set `PULUMI_BACKEND_URL` to the environment-specific bucket before running `pulumi up`. This is intentional — `Pulumi.yaml` cannot interpolate the stack name into the backend URL.

**infra/config.py**

```python
import pulumi

cfg = pulumi.Config()
gcp_cfg = pulumi.Config("gcp")

ENV: str = cfg.require("env")                    # "dev" | "staging" | "prod"
CLUSTER_NAME: str = cfg.require("cluster_name")  # "vici-dev" | "vici-staging" | "vici-prod"
REGISTRY_NAME: str = cfg.require("registry_name")
PROJECT_ID: str = gcp_cfg.require("project")
REGION: str = gcp_cfg.get("region") or "us-central1"
```

No environment conditionals in this file. All per-environment values come from stack config. The `or "us-central1"` fallback on REGION makes the field optional in stack configs while keeping the default explicit here.

Set PULUMI_CONFIG_PASSPHRASE for local dev (this is required for GCS backend secret encryption):

```bash
# Add to your shell profile (~/.zshrc or ~/.bashrc):

```

The same passphrase value must be stored as a GitHub Actions secret named `PULUMI_CONFIG_PASSPHRASE` before CI can run.
