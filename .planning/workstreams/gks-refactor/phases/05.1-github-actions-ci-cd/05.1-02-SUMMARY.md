## What Was Built

### cd-base.yml Two-Job Reusable Workflow (Task 1)

Rewrote `cd-base.yml` from a single-job workflow into a two-job reusable workflow per D-01:

**build job:**
- WIF auth via `google-github-actions/auth@v3` (id-token: write permission)
- `docker/setup-buildx-action@v3` for Buildx driver (required for GHA cache)
- Docker build+push via `docker/build-push-action@v6` with `cache-from: type=gha` and `cache-to: type=gha,mode=max`
- SHA-only tagging: `git rev-parse --short=7 HEAD` — no env-name tag (D-07)
- Conditional push: only runs when `inputs.command == 'up'` (skips for preview)
- Outputs `sha` for consumption by deploy job

**deploy job:**
- `needs: [build]` dependency on build job
- WIF auth (id-token: write permission)
- `astral-sh/setup-uv@v5` + `uv sync --frozen` for Pulumi Python dependencies
- `pulumi/actions@v6` with `cloud-url: gs://vici-app-pulumi-state-{stack}` (D-10: no PULUMI_ACCESS_TOKEN)
- `config-map: '{"vici-infra:imageTag": {"value": "...", "secret": false}}'` passes SHA to Pulumi (D-08)
- Health check step: 12 retries x 5s targeting `https://dev.usevici.com/health`, only for `stack == 'dev' && command == 'up'` (D-04)

New required input `gcp_project` eliminates runtime `gcloud config get-value project` call (Research Pitfall 6).

### Caller Workflow Rewrites (Task 2)

**cd-dev.yml:** Added `gcp_project: vici-app-dev`. Changed secrets from plural `GCP_WIF_PROVIDER_DEV`/`GCP_CI_SA_DEV` to singular `GCP_WIF_PROVIDER`/`GCP_CI_SA_EMAIL` per D-09/D-11.

**cd-staging.yml:** Removed `pull_request` trigger entirely (D-05 fix). Removed `preview` job. Single `deploy-staging` job calling `command: up` via `workflow_dispatch` only. Added `gcp_project: vici-app-staging`. Singular secret names.

**cd-prod.yml:** Added `gcp_project: vici-app-prod`. Singular secret names per D-09/D-11. Passes `environment: prod` to cd-base.yml for GitHub Environment approval gate (D-06/CD-03).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Quoted YAML `on` key to fix PyYAML boolean True parsing**
- **Found during:** Task 1 verification
- **Issue:** PyYAML `safe_load` parses unquoted `on:` as boolean `True:` key. Tests access `workflow.get("on", {})` which returns `None` since the actual key is `True`. `TestCDBaseStructure::test_cd_base_has_gcp_project_input` failed because `inputs` resolved to `{}`.
- **Fix:** Changed `on:` to `"on":` in all four workflow files. GitHub Actions accepts both forms — the quoted version is valid YAML and GitHub parses it correctly at runtime.
- **Files modified:** All four workflow files
- **Commits:** c25e3e3 (cd-base.yml), 1c3451a (caller workflows)
